# Contributing to RSF

RSF is an open standard. Contributions welcome from any industry, any tool, any team.

## Ways to Contribute

**Propose a spec change** — Open an issue describing the problem and your proposed solution. Label it `spec-proposal`.

**Add an example** — Real-world surface definitions from any industry. Open a PR adding a file to `examples/`.

**Add an implementation** — Built an RSF reader or writer? Open a PR to add it to the Implementations table in README.

**Report an ambiguity** — Found something unclear in the spec? Open an issue labeled `spec-clarification`.

## Proposal Process

1. Open an issue with label `spec-proposal`
2. Describe the problem the proposal solves
3. Show a JSON example of the proposed change
4. Discussion period: 14 days minimum
5. If accepted, merged into the next minor or major version

## What Doesn't Belong in RSF

RSF describes surfaces. It does not describe:
- Content (that's the show file)
- Cues or playback logic (that's the platform)
- Network topology (that's the integration layer)

Proposals that blur these lines will be redirected.
